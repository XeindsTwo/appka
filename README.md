# FitBook

**FitBook** - мобильное Android-приложение для фитнес-клуба. Система помогает вести клиентов, тренеров, расписание тренировок, абонементы и персональные тренировочные планы.

Проект разработан как локальное приложение без серверной части: данные хранятся в SQLite-базе на устройстве, интерфейс реализован на XML-разметке, логика написана на Java.

## Назначение системы

Приложение предназначено для автоматизации базовых процессов фитнес-клуба:

- регистрация и авторизация пользователей;
- разделение доступа по ролям;
- управление клиентами и тренерами;
- ведение расписания тренировок;
- оформление и просмотр абонементов;
- назначение клиентов тренерам;
- создание персональных планов тренировок;
- просмотр истории абонементов и пользовательского профиля;
- хранение контактной информации клуба.

## Роли пользователей

### Администратор

Администратор управляет основными данными клуба:

- добавляет и редактирует тренеров;
- регистрирует и редактирует клиентов;
- назначает клиентам тренеров;
- создает и изменяет расписание тренировок;
- управляет типами абонементов;
- просматривает списки клиентов, тренеров, расписания и абонементов.

Основной экран роли: `AdminActivity`.

### Тренер

Тренер работает со своими занятиями и клиентами:

- просматривает назначенные тренировки;
- видит список закрепленных клиентов;
- создает тренировочные планы для клиентов;
- добавляет упражнения, подходы, повторы и рекомендуемый вес.

Основной экран роли: `TrainerActivity`.

### Клиент

Клиент использует приложение как личный кабинет:

- просматривает доступные тренировки;
- записывается на занятия;
- смотрит свои записи;
- просматривает персональный тренировочный план;
- ведет замеры тела;
- просматривает прогресс;
- выбирает и оформляет абонемент;
- смотрит историю абонементов;
- открывает контакты клуба и профиль.

Основной экран роли: `ClientActivity`.

## Авторизация и защита доступа

В приложении реализована проверка доступа по ролям. После входа пользователь попадает только в тот раздел, который соответствует его роли.

Для защиты экранов используется `AuthGuard`:

- проверяет наличие активной сессии;
- сверяет пользователя с записью в SQLite-базе;
- проверяет роль пользователя перед открытием защищенного экрана;
- при ошибке доступа очищает сессию и возвращает пользователя на экран входа.

Это защищает приложение от ситуации, когда пользователь вручную попадает на экран другой роли.

## Основной функционал

- вход по логину и паролю;
- регистрация клиента;
- сохранение данных текущего пользователя;
- редактирование профиля;
- выход из аккаунта.

### Управление клиентами

- просмотр клиентской базы;
- поиск и фильтрация;
- регистрация нового клиента;
- редактирование данных клиента;
- назначение тренера;
- просмотр подробной карточки клиента;
- удаление клиента.

### Управление тренерами

- просмотр списка тренеров;
- добавление тренера;
- редактирование данных тренера;
- просмотр карточки тренера;
- удаление тренера.

### Расписание тренировок

- создание тренировок;
- выбор тренера, даты, времени, длительности и лимита клиентов;
- просмотр расписания;
- редактирование и удаление занятий;
- запись клиента на тренировку.

### Абонементы

- создание типов абонементов;
- редактирование абонементов;
- оформление абонемента клиентом;
- хранение активного абонемента;
- просмотр истории абонементов.

### Тренировочные планы

- тренер выбирает клиента;
- добавляет заметки к плану;
- добавляет упражнения;
- указывает количество подходов, повторов и рекомендуемый вес;
- клиент видит назначенный план в своем кабинете.

### Замеры и прогресс

- клиент может сохранять антропометрические данные;
- система отображает прогресс и количество завершенных тренировок;
- данные сохраняются локально в SQLite.

## База данных

SQLite-база создается в `DatabaseHelper`. Основные таблицы:

- `users` - пользователи, роли, логины, пароли и контактные данные;
- `trainers` - информация о тренерах;
- `client_assignments` - связь клиента и тренера;
- `schedule` - расписание тренировок;
- `bookings` - записи клиентов на тренировки;
- `membership_types` - типы абонементов;
- `memberships` - абонементы клиентов;
- `membership_applications` - заявки и параметры оформления абонемента;
- `workout_plans` - тренировочные планы;
- `plan_exercises` - упражнения в плане;
- `workout_results` - результаты тренировок;
- `anthropometry` - замеры тела клиента.
